# Acceptance Criteria

## Fase 1

- [x] Repo bevat duidelijke projectintro.
- [x] MVP-scope staat vast.
- [x] Niet-v1 scope staat vast.
- [x] Publieke intakevelden zijn gespecificeerd.
- [x] Prioriteiten zijn gespecificeerd.
- [x] Kantooruren zijn vastgelegd.
- [x] Uploadregels zijn vastgelegd.
- [x] Dashboardroute en datamodel zijn beschreven.
- [x] Designrichting op basis van referentie is beschreven.
- [x] Wireframe voor publieke supportpagina bestaat.

## Fase 2 — Backend klaar wanneer

- [ ] Ticket kan aangemaakt worden via backend call.
- [ ] Ticketnummer wordt automatisch gegenereerd.
- [ ] Validatie weigert ongeldige input.
- [ ] Uploads worden veilig opgeslagen en gekoppeld.
- [ ] Dashboardquery toont tickets.
- [ ] Status update werkt.
- [ ] Interne comment werkt.

## Fase 3 — Dashboard klaar wanneer

- [ ] `/support` zichtbaar is in sidebar.
- [ ] Ticketoverzicht toont echte tickets.
- [ ] Filters werken.
- [ ] Volledige ticketkaart/rij is klikbaar.
- [ ] Detailpagina toont alle ticketinfo.
- [ ] Status wijzigen werkt vanuit UI.
- [ ] Interne notities zijn zichtbaar in tijdlijn.
- [ ] Bijlagen zijn downloadbaar/bekijkbaar.

## Fase 4 — Publieke site klaar wanneer

- [ ] `support.stinismedia.com` laadt.
- [ ] Formulier werkt op desktop en mobiel.
- [ ] Formulier toont duidelijke validatiefouten.
- [ ] Ticket komt in dashboard.
- [ ] Klant ziet ticketnummer.
- [ ] Uploads werken.

## Fase 5 — Notificaties klaar wanneer

- [ ] Klant ontvangt bevestigingsmail.
- [ ] Stinis ontvangt interne melding.
- [ ] Noodgevalmelding is duidelijk herkenbaar.

## Fase 6 — Live klaar wanneer

- [ ] Productie env vars staan correct.
- [ ] DNS staat correct.
- [ ] Smoke test met normaal ticket geslaagd.
- [ ] Smoke test met noodgeval geslaagd.
- [ ] Smoke test met bijlage geslaagd.
